Spring Tool Suite


如果想將〈Spring Boot CLI〉的 hello 專案匯入至Eclipse之中，可以執行選單「File/Import…」，在「Import」對話方塊中選擇「Gradle/Existing Gradle Project」，然後在後續操作中選擇 hello 資料夾完成匯入。

想要執行 bootRun 任務的話，可以於「Gradle Task」頁籤中，展開「hello/application」，於「bootRun」上按右鍵執行「Run Gradle Tasks」就可以了：

若要停止執行，「Gradle Task」頁籤右方有個紅色按鈕，按下就可以取消任務。

另一種方式，就是直接找出 @SpringBootApplication 標註之類別，在上面按右鍵執行「Run as/Java Application」就可以了。

在開發 Spring Boot 專案上，Spring 官方提供了 Spring Tool Suite，可以在既有的 Eclipse，透過「Help/Eclipse Marketplace」來安裝 Spring Tool Suite，或者是直接下載 Spring 官方基於 Eclipse 打造好的 Spring Tool Suite。

以 Spring Tool Suite 4 為例，在下載 zip 檔案並解壓縮之後，可以在資料夾中看到 SpringToolSuite4 執行檔，直接執行就可以啟動 Spring Tool Suite，由於是基於 Eclipse，一樣也是先選擇 Workspace 資料夾。

Spring Tool Suite 4 中可以使用 Buildship 3.x，不過建議更新一下（避免在使用新版 JDK 卻發生找不到 JDK 版本的訊息之類），可以透過「Help/Eclipse Marketplace」搜尋 Buildship 來更新 Gradle 官方的 Buildship plugin。

如果想要建立新的 Spring Boot 專案，可以執行選單「File/New/Project…」，在「New Project」對話方塊中，選擇「Spring Boot/Spring Starter Project」，按下「Next >」後，可以看到在底下的畫面進行選項設定：

在按下「Next >」之後，可以看到有許多 Starter 專案的選擇，可以透過「Available:」底下的文字框搜尋想要的選項。

接著可以直接按下「Finish」完成專案建立，你可以試著在其中實作〈Spring Boot CLI〉的範例，體會一下在 Spring Tool Suite 中進行程式撰寫、設定時的便利性，想要執行專案，除了上述「Run Gradle Tasks」的方式之外，也可以直接在專案上按右鍵，執行「Rus as/Spring Boot App」。
